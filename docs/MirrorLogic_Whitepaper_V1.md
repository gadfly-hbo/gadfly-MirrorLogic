## 第一章：学术理论基础 (全方位支持)
灵犀镜像的核心算法引擎是奠基在行为学与组织心理学基础上的量化指标：**PLSE 分析面板**。
传统的 MBTI 等模型旨在描述性格特征，而 **PLSE 的设计初衷是为了高压环境下的“防御阻击”与“说服开关”**。

1. **P (Power / 控制欲与等级)**
   - **理论根基**：麦克利兰的“成就动机理论”中的核心需要——权力动机（nPow）。
   - **实战意义**：决定了对方是“命令者”还是“配合者”。针对极高 P 的对象，系统判定只能用低姿态、提供客观选择题的沟通框架，绝对不能采用剥夺其决策权的命令式话术。

2. **L (Logic / 逻辑与利益导向)**
   - **理论根基**：卡尼曼《思考，快与慢》中纯理性的“系统2”，以及双系统处理模型。
   - **实战意义**：决定了对方是否吃数据和金字塔原理。针对强逻辑目标（如研发、理科CEO），系统将禁止一切软弱的情绪词汇，转而使用 ROI、数据报表与结论先行。

3. **S (Stability / 安全冗余与风险厌恶)**
   - **理论根基**：前置理论（Prospect Theory）的损失厌恶与调节焦点。
   - **实战意义**：识别其追求“不犯错”的目标层级。遇到满值 S 的保守体制内高管，策略引擎会放弃“画大饼”，转而用责任分割协议、风险对冲与冗余条款打消其防御心。

4. **E (Empathy / 共情与情绪链接)**
   - **理论根基**：人本主义心理学体系与情商理论（EQ）。
   - **实战意义**：高 E 者需要非暴力沟通（NVC），不讲对错，只论感情。系统此时会强制加载情感隔离垫——即“看见对方的情绪”，作为破冰钥匙。

## 第三章：开发复盘沉淀 (技术与经验教训)

在本期 V1.0 的冲刺开发中，我们遭遇了诸多技术断层，并完成了一次架构的深层次重组：

### 1. 经验：彻底解决大语言模型 (LLM) 流式输出结巴
* **困境**：在引入 Deepseek 流式返回数据时，页面呈现极度诡异的文字重影复制与跳词。
* **排雷与测试**：我们一度更换 API Key (如 Kimi、Claude 测试)，甚至在后端加入极其复杂的正则表达式（如利用 `[^\x00-\xff]` 和标点符号查重）。但最后证实这一切都是掩耳盗铃。
* **终局**：真正导致该 Bug 的，是 React 18 中的 `StrictMode` 与 State Mutation (违规范修改对象引用) 的撞车。系统进行两次渲染，由于旧引用被污染，字被拼装了两次。
* **沉淀**：已总结出 `react_strictmode_llm_streaming.md` 通用最佳实践 Skill。今后不可变数据模式必须贯彻到极深的数据解构中。

### 2. 经验：UI 布局体验的克制
* **痛点**：由于策略工作坊内 AI 输出大段高质量的解析，旧版的卡片由于局限在 `flex-box` 加上过多的 Padding 与狭小的 TextArea 中极其难看，换行也极其随意。
* **动作**：精简重组界面。对过长的列表页面改写了**“时间轴动态分组切割分类并加入手风琴折叠”**，极具现代感与实用性。
* **沉淀**：大语言模型时代，长文本（Token）视觉疲劳无法避免，必须提供诸如 **全屏放大功能、一键复制功能以及区块化锚定高亮组件** 才能减轻用户负担。

### 3. 经验：逻辑中控（The Engine）代替裸聊
* **反思**：最初的沙盘就像市面上各种裸着 API 接口的聊天机器人一样没有边界感。
* **重构**：我们加入了“中间拦截器”（Intermediate Layer），通过硬编码识别 PLSE 值，将“强词汇法则”封装入提示词，再送去给 LLM（如强制加上 FBI 谈判理论）。这样出来的文案脱胎换骨，极具杀伤力。

> 至此，MirrorLogic 1.0 的底层逻辑骨架构筑圆满，前端热启动和冷启动流程畅通。后续向 2.0 迈进时，可全面开启更细致的人工逆向剖析与全局问卷导入。
